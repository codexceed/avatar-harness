avatar-harness · core-library readiness

From a decoupled engine to an
importable, extensible core.

The vision: the harness is the minimum reusable engine for building bespoke agentic apps. The model and any UI/UX layer (the planned interactive REPL is just one) live outside that boundary and build atop it. This page maps where the code stands today and what "robust core" requires next.

The hard part is done; the easy-sounding part isn't. The abstractions are ahead of the runtime enforcement — though Phase 2.5 shows enforcement can catch up (secret-safety is now solid).

The boundary, as it stands

01Three layers — one of them leaks

Structurally the split is real: UX talks to the engine only through event subscribers and (today) one CLI assembler; the model sits behind an interface. But the core still imports model concerns, and the UX boundary is the only one that's truly clean.

Subscribes to observation events; should call a core facade. Today there is no facade — only cli.run_agent().

02What's solid, what's soft

Every claim below was read out of src/avatar_harness/ and independently re-checked by Codex. The foundation (DI, headlessness, open tools) is genuinely there — graded honestly.

STRONG — true today PARTIAL — true but leaks GAP — missing for the vision DEAD-WIRED — declared, not enforced

STRONG Headless engine

No terminal I/O anywhere in the engine. UX is purely an event subscriber.

verified · zero print/argparse/input/rich outside cli.py · caveat: ArtifactManager.render is presentation-in-core

STRONG Open tool registry

Add a bespoke tool with a callable + one register() — no core edit.

ToolHandler, ToolDefinition, ToolRegistry.register · defaults are mere assembly in default_registry()

STRONG No globals · run-scoped deps

Everything a tool needs arrives via an explicit dataclass. Replayable.

RunDeps (deps.py) passed into every handler by ToolRuntime.execute

STRONG Programmatic config

Env-aware, not env-locked — constructible with kwargs for embedders.

HarnessConfig(BaseSettings) · injected by both run_agent and main

STRONG · new in 2.5 Secret-safety, enforced consistently

The model where enforcement did catch up: a sensitive-path denylist refused on read & patch and excluded from search — two-layer (gate up front + workspace re-check on the resolved path, so a symlink can't launder a secret).

canonical path_is_sensitive() at all 3 sites (gate · workspace backstop · search_repo filter) — can't drift per tool · overridable sensitive_path_globs

PARTIAL Model behind an interface

The ABC is real, but model concerns live inside the core and the default prompt isn't neutral.

ModelClient ABC ✓ · but openai is a hard dep, _SYSTEM_TEMPLATE in core, prompt hardcodes "READ-ONLY"

PARTIAL Constructor DI

All collaborators injected — but most seams are concrete classes, not protocols. Swapping = subclassing.

AgentRunner.__init__ kw-only · only ModelClient is abstract; ContextBuilder/Verifier/Policy/Emitter are concrete

GAP No public API / facade

Package root exports only __version__ (+ a stale "not started" docstring). Consumers deep-import internals; no __all__. The only assembler is the CLI.

__init__.py bare · embedders can build AgentRunner directly — what's missing is curation, not capability

DEAD-WIRED Phase gating

Phase advertises a tool subset to the model but is never enforced — and never even advances.

state.phase is never assigned in src/ · ToolRuntime.execute does no phase check

GAP Tool-failure isolation

A third-party tool that raises crashes the whole run — the handler call is unguarded.

ToolRuntime.execute catches input validation only; tool.handler(...) is uncaught

DEAD-WIRED Budgets & cancellation

Config advertises wall-clock / context budgets and a cancellation token the loop never honors.

max_wall_clock_seconds/max_context_tokens unreferenced · CancellationToken created, never consumed

GAP Closed verification contract

The most domain-specific component is the least extensible — opposite of the open tool registry.

Verifier.verify = if/elif on task_kind → NotImplementedError; task_kind a closed Literal

PARTIAL Workspace = git

Editing/verification assume a git tree. A real reuse constraint for non-git backends. Also: search_repo bypasses Workspace.run.

apply (git apply --index) · clean-start (git status) · baseline (rev-parse) · diff (git diff)

The proposal · twelve control axes

04Now → Proposed

None of these require re-architecting — every change is additive, which is the strongest signal the foundation matches the vision. The throughline: turn declared abstractions into enforced runtime guarantees, then package the seams.

Public API surfaceGAP
Now

__init__.py exports only __version__ + a stale docstring. No __all__; consumers deep-import internals.

→
Proposed

Curated __all__: AgentRunner, TaskState, ToolDefinition, ToolResult, RunDeps, ModelClient, Workspace… — a documented, stable import surface.

Assembly & embeddingGAP
Now

The only wiring lives in cli.run_agent() (UX layer), hardcoding the default registry / model / verifier.

→
Proposed

A core-side Harness / Session facade with every part overridable. The CLI becomes a caller — drawing the boundary in code, not by convention.

Phase gatingDEAD-WIRED
Now

state.phase is never assigned and never enforced; the phase-active toolset is a prompt hint only.

→
Proposed

Runner advances phase; ToolRuntime enforces the active set — an out-of-phase call is blocked and fed back, not silently executed.

Tool-failure isolationGAP · highest leverage
Now

ToolRuntime.execute doesn't wrap the handler — a raising third-party tool unwinds the loop.

→
Proposed

Handler wrapped: exceptions become ToolResult(success=False), split into model-correctable vs system failure (§10) — a bad tool can't crash the run.

BudgetsDEAD-WIRED
Now

max_wall_clock_seconds and max_context_tokens are configured but never referenced by the loop.

→
Proposed

Loop honors wall-clock + context budgets, mapping exhaustion to incomplete — matching the documented bounding contract (§5).

CancellationDEAD-WIRED
Now

CancellationToken is created in the CLI and consumed nowhere — the §18 cancel-race reuse isn't hooked up.

→
Proposed

Token checked at turn/tool boundaries; powers per-tool timeout and interactive interrupt (ESC → add_feedback).

Model boundaryPARTIAL
Now

openai is a hard dep; the prompt contract sits in core; the default prompt hardcodes "READ-ONLY investigation".

→
Proposed

Prompt + provider live behind the adapter; openai becomes an optional extra; the default is kind-aware or neutral.

Verification contractGAP
Now

Closed if/elif over three task_kinds; unknown kinds raise. Custom contracts require forking the verifier.

→
Proposed

A verifier-strategy registry keyed by contract (mirroring the tool registry); the three built-ins ship as defaults; bespoke gates plug in.

Drive surfaceGAP
Now

Only run(state) — run-to-completion. Mid-run injection is limited to the permission callback + subscribers.

→
Proposed

A step() / driver / generator surface that yields at decision & approval points — enabling suspend/resume for servers and human-in-the-loop.

ConcurrencyDELIBERATE CUT
Now

Fully synchronous — no async anywhere (an intentional MVP simplification).

→
Proposed

Decide deliberately: async core (or sync-core + async step) for concurrent multi-session servers. The one gap that's costly to retrofit late.

Workspace backendPARTIAL
Now

Hard git-backed (apply / clean-start / baseline / diff all shell out to git).

→
Proposed

Workspace behind a protocol; git is one implementation; non-git backends (in-memory, remote, snapshot) become possible.

Internal consistencyPARTIAL · improved in 2.5
Now

Fixed in 2.5: the sensitive-path matcher is now canonical across all sites, and TaskState.decisions is populated by the runner's ledger. Remaining: search_repo still shells out directly (not via Workspace.run); events are raw dicts; current_plan is still unwritten.

→
Proposed

Route the last tool (search_repo) through Workspace; a typed event schema consumers can depend on; prune or populate the last dead field (current_plan).

Register a bespoke tool — no core editregistry: today · @tool sugar: proposed
from avatar_harness import tool, ToolResult

@tool(name="grep_logs", phases={"investigating"}, tier=0)
def grep_logs(args, deps) -> ToolResult:
    out = deps.workspace.run(["rg", args.pattern, "logs/"])
    return ToolResult(success=True, content=out.stdout, summary="log hits")

harness.tools.register(grep_logs)
Proposed — verification contracts become an open registry (like tools)proposed
from avatar_harness import verifier_contract, CheckResult

@verifier_contract("deploy")              # a bespoke contract beyond edit/investigate/test
def verify_deploy(state, ws) -> list[CheckResult]:
    out = ws.run(["./scripts/smoke.sh"])
    ok  = out.exit_code == 0
    return [CheckResult(name="smoke", kind="required",
                        status="pass" if ok else "fail",
                        evidence=out.stdout[-500:])]

harness.run("ship the hotfix", task_kind="deploy")
Proposed — subscribe to events; act via control methods (two planes)proposed
# Observe via events() (never blocks the loop); act via explicit control methods.
async with harness.session("refactor the parser") as session:
    task = asyncio.create_task(session.run(task_kind="edit"))

    async for event in session.events():           # OBSERVATION plane
        tui.render(event)
        if isinstance(event, ApprovalRequested):     # CONTROL plane — explicit, awaited
            await session.resolve_approval(event.approval_id, await tui.ask(event))
        if isinstance(event, AgentEnd):
            break
    artifact = await task

# ESC handler → control plane, separate from the event stream:
await session.cancel("user pressed ctrl-c")      # trips token → records add_feedback
Proposed — durable resume: continue a crashed / paused runproposed · the frontier
# The journal is a write-ahead replay substrate, not just a log.
session  = await harness.resume(session_id)       # replay journal → rehydrate TaskState
artifact = await session.run()                    # continues where it left off:
                                              #   reads reused · patches NOT re-applied
                                              #   pending approval re-offered (same id)

Where this sits vs. the field

06SOTA alignment — and the durable frontier

Measured against the prevailing 12-Factor Agents conventions, the harness is already on the right side of most — its verification-terminates-the-loop invariant is ahead of frameworks that let the model self-certify. The one conspicuous miss is launch / pause / resume, which is precisely the 2026 frontier: durable execution. That's the committed next horizon; everything past it is deliberately deferred.

✓ Own control flow ✓ Own context window ✓ Tools = structured outputs ✓ State-as-truth / reducer ✓ Small, typed tools ◑ Own your prompts (leaks) ◑ Observability (no OTel yet) ✗ Launch / pause / resume → durable execution
Scope line. Committed frontier: durable execution — resumable runs over the event journal. Deliberately deferred past it: MCP interop, a middleware pipeline, and graph / declarative topology — revisited only if the goal widens beyond a single verifiable coding loop.
Async events + durable execution · three planes around the loop
observation (out) control (in · blocks) durable replay

Sequenced by leverage, not by section number

07Prioritized hardening path

Order chosen so a third party hits hardened seams first. Enforcement before packaging: a clean API over a loop that a bad tool can crash, or a budget that silently no-ops, sells a promise the core doesn't keep.

Isolate tool failures

Wrap tool.handler in ToolRuntime.execute; map exceptions to a failed ToolResult with the model-correctable / system split. The seam every third-party tool hits first.

small · high value

Make phase gating real

Advance state.phase in the runner and enforce the active toolset at execution — turn the advertised control axis into an enforced one.

medium

Honor budgets & cancellation

Wire wall-clock / context budgets to incomplete; consume the CancellationToken at turn/tool boundaries. Stop advertising unkept capabilities.

medium

Public API + Harness facade

Curate __all__; lift assembly out of the CLI into a core facade. Forces the boundary into code and gives the CLI as the first real consumer of your own API.

medium

Neutralize the model boundary

Move the prompt contract behind the adapter; make openai an optional extra; drop the hardcoded "READ-ONLY" default. Puts the model genuinely outside the core.

medium

Async engine + typed event bus

Make AgentRunner.arun() async (sync run() wraps it); replace raw-dict events with a typed union over a non-blocking bus (bounded per-subscriber queues; the journal is the one lossless, awaited sink). TUI subscribes via events(); approval/cancel stay explicit control methods. Open the verifier into a contract registry alongside.

larger · design-weighted

Durable execution — the SOTA frontier

Promote the journal to a write-ahead replay substrate: per-turn TaskState checkpoints, write-ahead intent before any side effect, and semantics-aware resume (reuse logged reads; never re-apply a patch; resume into a pending approval). Pause/resume survives a restart. Scope ends here — MCP, middleware, graph deferred.

larger · frontier
Provenance. Strengths and gaps were read directly from src/avatar_harness/ and independently cross-validated by Codex (gpt-5.4, reasoning effort xhigh, read-only). Updated 2026-06-08 for Phase 2.5 (PR #5 · context & secret hardening). What changed: a sensitive-path denylist is now enforced two-layer (permission gate + workspace resolved-path backstop) through a single canonical path_is_sensitive() matcher used at every site, and the TaskState.decisions ledger is now populated. The four control-axis findings — phase not advanced/enforced, budgets & cancellation unconsumed by the runner, and the unguarded tool handler — are unchanged, re-verified against source at 115/115 green. So the thesis still holds for the control plane; secret-safety is the standing counter-example.
Bottom line
The UI boundary is genuinely clean and the engine is properly decoupled — the architecturally expensive properties. The remaining work is to enforce what's already declared and package the seams, neither of which undoes the foundation.
