CLI entry point.

`main()` drives the real loop via `run_agent`. The CLI stays a thin shell over the
loop — wiring components and event subscribers, nothing more.

## Functions
### `main(argv: list[str] | None = None) -> int`

CLI entry point: parse args, wire the loop, run the task, render the result.
### `run_agent(task: str, *, config: HarnessConfig, emitter: Emitter, model_client: ModelClient | None = None) -> TaskState`

Run the read-only investigate loop over `task` (Phase 1).
