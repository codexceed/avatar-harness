## Mode (novel — bucket A, harness/scaffold; provider transport)

**A NUL/hung provider reply truncates the run.** Under concurrent load (ADR-0026), `minimax/minimax-m3` via OpenRouter held connections ~5–6 min and returned a ` ` (NUL) body. The harness has **no request timeout** and **no transport-layer retry**, so the hang ate the wall-clock budget and the cell ended after a single turn as `outcome=incomplete`. The NUL — a successful HTTP 200 the SDK won't retry — was misrouted through the **model parse-retry** (`max_parse_retries`), which re-prompts the model: the wrong layer for a transport failure, and a §16 violation. This is the harness/scaffold defect that the run-C "regression" turned out to be (not a model or code-quality regression). Proposed as catalog **A9**.

## Evidence of the precise mechanism

- **Concurrent run** (`eval_run_20260620T142752Z`): 4 minimax cells hit `decision_error raw=" "` after 297–364s hangs; their end-timestamps converge at ~14:38:1x → simultaneous provider degradation. 3 failed; `modify-existing/seed0` and `seed3` ended after **one** `list_files` turn → `incomplete`.
- **Serial control** (`eval_run_20260620T161006Z`, `--concurrency 1`): minimax **20/20, 0 NULs**, turns/cell mean 5.0. minimax pass@1 across conditions: **0.90 → 0.85 → 1.00**. The failures were transport, not capability.
- **Root cause** (`model_client.py:484`): `OpenAI(...)` built with no `timeout=`/`max_retries=`; SDK default timeout ≈ `max_wall_clock_seconds` (600) → one hung call consumes the whole budget. NUL = HTTP 200 → SDK never retries → `parse_decision(" ")` → `DecisionParseError` → parse-retry **re-prompts the model** (`:530`).

## Remediation (code_logic × global → adr_only) — see ADR-0028

Split transport from parse failures: **(R1)** bound every request with `request_timeout_seconds=90` (≪ 600s wall clock); **(R2)** classify empty/whitespace/all-NUL bodies as a new `EmptyResponseError` (transport), not `DecisionParseError`; **(R3)** retry transport with **backoff + jitter** (`transport_max_retries=3`), re-issuing the same `messages` (no context pollution); **(R4)** on exhaustion surface a **system failure** (§16) and emit a `transport_retry` event so the journal stops mislabeling NULs as `decision_error`. `(R5)` streaming with an idle-timeout is a deferred latency optimization.

Because this is an **always-on transport change → `blast_radius = global`**, `ChangeProposal.route()` returns `adr_only`: it lands as **ADR-0028**, promoted to Accepted only after the offline scripted-client suite is green (the deterministic proof — a real provider NUL can't be reproduced on demand) and, optionally, a confirmatory `--concurrency 4` minimax re-run shows `transport_retry` events with **zero one-turn-incomplete cells**. It does **not** touch any spec/probe/fixture/verifier/scoring → `touches_grader = false`. Pairs with a separate eval-runner jitter / per-provider concurrency-cap change that removes the trigger.
