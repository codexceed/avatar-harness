# DECISIONS ‚Äî design decision log

**The running record of *why* the build is shaped the way it is.** Each entry is a dated decision: the choice made, the alternatives weighed, the trade-off accepted. `PROGRESS.md` tracks *how far* the build has gotten (checklists); `HARNESS_DESIGN.md` is the *what/why* spec; `ARCHITECTURE.md` is the current system map. This file is the chronological ledger of decisions taken along the way.

> **When to add an entry:** whenever a **major design decision** is made ‚Äî a chosen approach, a rejected alternative, a scope cut, a non-obvious trade-off, or a load-bearing clarification. Append a dated bullet (`- **YYYY-MM-DD** ‚Äî ‚Ä¶`) and keep the *why* and the alternatives, not just the *what*. Routine implementation that merely follows an existing decision needs no entry.

- **2026-06-09** ‚Äî **`write_file` ‚Äî first-class file creation (ADR-0003 Option B, implemented).** Creation gets a plain-content transport; the diff costume comes off the no-anchor case. Design choices: **(1) The Workspace stays the single mutation chokepoint** ‚Äî a new `Workspace.write_file(path, content, overwrite=False)` does confinement (`_resolve`) + the denylist (`_assert_not_sensitive`) + write + **stage** (so the new file appears in `diff()` ‚Üí artifact ‚Üí verifier, matching `apply_patch --index` and the `run_command` mutation capture); the tool is a thin handler over it. **(2) Modification stays diff-anchored**: an existing target raises `FileExistsError` ‚Üí a model-correctable result that *names `apply_patch`* as the right tool, unless `overwrite=true` is explicit ‚Äî so the clean-apply staleness invariant (inspect-before-edit emerges from `git apply --check`) cannot be bypassed casually. **(3) Tier-1, declared-path gated**: `paths=lambda args: (args.path,)` hands the target to the central gate (confinement + denylist for free, the Phase-2.5 unification working as designed); tier 1 means the edit-intent bootstrap (¬ß2.6) advertises and admits it from `investigating` on edit-shaped tasks, and an `investigate` task still cannot mutate. **(4) Lives in `tools/edit.py`** beside `apply_patch` (the mutating-tier module), registered in `default_registry`. - **2026-06-09** ‚Äî **Native tool-calling is the default decision transport (ADR-0003 Option A, accepted & implemented).** The maintainer took A *first* (the ADR recommended B-then-A): the ¬ß6 decision now rides the provider's function-calling channel instead of a hand-written JSON envelope. Mechanics: `build_tool_schemas` sends every phase-admitted tool as a function (its real pydantic `input_schema`, not prose) plus `final_answer`/`ask_user` as functions ‚Äî so every decision shape is a provider-validated structured call, and a 200-line patch can no longer die in JSON escaping (the dogfood failure class). Design choices: **(1) Compatibility is automatic, not configured** ‚Äî a reply with no `tool_calls` (an endpoint that ignored `tools=`) falls through to the legacy `parse_decision` on its content, so "OpenAI-compatible" endpoints keep working with zero setup; `AVATAR_NATIVE_TOOL_CALLS=false` additionally forces the legacy protocol verbatim for endpoints with actively broken tool-call support (the escape hatch; default on). **(2) `tool_choice` is left unset** (`auto`) ‚Äî `required` is unevenly supported across compat endpoints, and the content-fallback covers a prose reply anyway. **(3) Retries keep the ¬ß18 invariants**: a malformed-arguments attempt is retried in-conversation with valid `tool_call_id` pairing (assistant tool-call message answered by a `role="tool"` reply) and lands on the `retry_trace` like any malformed decision, so the observability fix composes. **(4) The native system prompt drops the JSON-envelope contract** (that instruction conflicted with tool-calling) while staying kind-aware; `build_messages(native_tools=)` keeps the legacy prompt byte-identical for the fallback path. **(5) First tool call wins** when a model emits several (¬ß6: exactly one action per turn). `ModelClient` ABC, runner, tools, verifier: untouched ‚Äî the change is adapter-internal (Principle A). One Phase-1 test's assertion moved with the contract (tool advertisement now asserted in `tools=`, not prompt prose). ADR-0003 updated (Accepted; B open). 8 tests; 257/257 green.
- **2026-06-09** ‚Äî **Decision-parse failures are observable end-to-end; the cockpit renders the verifier's verdict** (dogfood post-mortem, journal `events/0ad6c3fe‚Ä¶jsonl`). The incident: on a "write a script" goal the model's `apply_patch` attempts repeatedly died as `DecisionParseError`s *inside* `OpenAIModelClient.decide()`'s retry loop ‚Äî leaving no journal event, no `TaskState` evidence, and no action-ledger entry ‚Äî so the model (whose context showed only successful reads) looped for 24 turns, hallucinated "I already applied the patch", and the cockpit showed `‚ñÝ success` even though the verifier had failed `diff_present` (conversational mode delivered the reply, 3.2d, and the promised ‚úì/‚öÝ badge had never been implemented). Decisions: **(1) Annotate, don't swallow.** The in-client retry loop stays (it preserves the raw malformed reply in-conversation, the best correction signal) but now annotates the surviving decision with a `retry_trace` (`DecisionRetryNote(error, raw‚â§2k)`); the **runner** ‚Äî the one mutator ‚Äî turns each note into `decision_error` evidence (the model sees its own failed attempts next turn) and a typed `DecisionError` event (`recovered=True/False`; the exhausted path now publishes to the bus too, not just the legacy emitter). *Rejected alt:* threading an event sink into `ModelClient` ‚Äî the client stays a pure adapter; diagnostics ride the return value. **(2) `retry_trace` is a harness-owned channel**: `parse_decision` clears it, so a model emitting the field can't plant fake history. **(3) The retry nudge asks for the SAME intended action**, not "a valid decision" ‚Äî the old wording invited the downgrade-to-a-read cope that fueled the loop. **(4) The cockpit renders the verdict** (`‚úì/‚öÝ verification ‚Ä¶` transcript line + a `verify:` status-bar badge) and `‚Ü© malformed decision` lines ‚Äî in conversational mode `outcome` alone reads "success", so without the badge the human "terminal authority" was deciding blind. The structural half (why a new-file patch can't survive JSON-escaped-diff transport) is **ADR-0003 (Proposed)**: `write_file` now, native tool-calling later. 7 tests; 249/249 green.
- **2026-06-09** ‚Äî **Interactive runs are journaled: one write-ahead journal per REPL sitting** (post-MVP gap-fix). The `--interactive` cockpit streamed events to the TUI transcript only ‚Äî nothing reached disk, so an interactive run was unreplayable after the fact (the exact opposite of batch, which has had `events/<session_id>.jsonl` since Phase 0). Decisions: **(1) Persist via the privileged `JsonlEventJournal` on the `EventBus`, not the `EventLog` emitter subscriber.** The journal is write-ahead and lossless (committed + flushed *before* the lossy fan-out), so a TUI crash or a shed `model_update` never loses the record; wiring `EventLog` as another subscriber would have made the durable record exactly as droppable as the display. The socket already existed (`Session(journal=‚Ä¶)`, built in Lane 1 and never used) ‚Äî this is wiring, not new machinery. **(2) One journal per REPL *sitting*, not per goal.** Each goal runs as its own `Session` (own `session_id`, own bus), so the naive wiring yields one file per goal; instead `ReplSession(journal=‚Ä¶)` holds a single journal and threads it into every per-goal `Session` **shared by reference, exactly like `grants`** ‚Äî "one sitting, one log," matching the batch one-file-per-invocation layout. Goals stay distinguishable inside the file by their per-goal `session_id`. The mechanism that makes sharing safe: each goal's `bus.close()` closes the journal handle, and `JsonlEventJournal.append` *reopens on demand* (append mode) for the next goal. **(3) Same managed layout + flags as batch**: `events/<repl-session-id>.jsonl`, `--log PATH` honored, `latest.jsonl` pointer maintained only for the managed layout (swung after the cockpit exits ‚Äî the file exists from launch, so it never dangles). **(4) `Harness.session()` gained the same `journal=` kwarg** (SDK parity; default `None` keeps the in-memory-only foundation behavior). Durable *resume* (checkpoint/replay semantics) remains 3.3 ‚Äî this is only "stop throwing away the flight recorder," and 3.3 will replay the very record this starts writing. 6 tests; 248/248 green.

- **2026-06-09** ‚Äî **3.2d ‚Äî conversational verification authority** (¬ß23.5, ADR-0002 D7). The verifier **always runs and always reports**; only its *authority* shifts with who's in the loop. Implemented as a single `conversational: bool` flag on `AgentRunner` (default `False`) that branches `_averify` *after* the verifier has run + emitted its events + appended its result ‚Äî so the "verify always runs" guarantee is structural, not mode-dependent. Decisions: **(1) Conversational failure resolves to `outcome="success"`, verdict advisory.** The chosen fork (over "verdict-as-outcome, no repair"): in chat mode the engine's job is to produce + report a reply, not to pronounce pass/fail ‚Äî "the human is terminal authority." The real verdict lives on `verifier_results[-1]` for the cockpit to render as a ‚úì/‚öÝ badge; `outcome` means "the turn delivered a reply." This is the most faithful reading of "does not block the reply." Accepted trade-off: a conversational run with a failed check reports artifact status `success` (the verification section still shows the failed checks) ‚Äî acceptable because conversational mode is explicitly *not* a CI gate. **(2) Default is strict at the engine/facade, conversational at the REPL.** `AgentRunner`/`Harness.run`/`arun`/`session()` default `conversational=False`, so all 229 prior tests and the SDK's batch semantics are untouched; `ReplSession(harness, *, auto=False)` is the one place that defaults *conversational* (builds runners with `conversational=not auto`) ‚Äî because the REPL is the interactive driver ¬ß23.5 describes. `--auto` (CLI, 3.2e) sets `auto=True` ‚Üí strict. **(3) No new `outcome` enum value, no repair in conversational mode** ‚Äî the flag only changes disposition (deliver vs. repair-loop), keeping `runner.py` close to the ¬ß5 shape. - **2026-06-09** ‚Äî **Lane 1 ‚Äî bounded `EventBus` + privileged write-ahead `JsonlEventJournal`** (ADR-0001), built sequentially.
